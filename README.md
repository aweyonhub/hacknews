## 快速开始（只需 2 步）

```bash
# 第一步：收集新闻
# 向 AI 发送：
#   "搜索 Hacker News 上 4月7日 到 4月13日 的热点新闻 每天 要有10条"
#
# → AI 返回带编号的列表，你查看后决定哪些需要深度解读

# 第二步：深度解读 + 归档（一站式完成）
# 向 AI 发送（三选一）：
#   A. "对 #1.3、#2.7、#3.5 这几篇做深度解读"     ← 指定编号
#   B. "对认知暗森林、Copilot广告事件做深度解读"     ← 指定标题
#   C. "对所有文章做深度解读"                        ← 不指定 = 全部
#
# → AI 自动完成：撰写解读 → 归档文件 → 生成周度索引 → 更新README → 全项目检查

# （可选）后续清理
# 向 AI 发送：
#   "清理部分文档"
# → 删除 README 中不需要的索引条目和对应文章文件（week.md 列表不动）
```

## 🌐 部署（可选）

项目支持通过 **Hugo + GitHub Pages** 一键部署为静态网站：

| 功能 | 说明 |
|------|------|
| 静态站 | [PaperMod](https://github.com/adityatelange/hugo-PaperMod) 主题，简约美观，响应式设计 |
| 全文搜索 | Fuse.js 客户端模糊搜索，离线可用，支持标题和内容搜索 |
| 自动部署 | push 到 main 分支即触发 GitHub Actions 自动构建和发布 |
| 自动同步 | 自动同步根目录下所有 markdown 文章，无需手动操作 |
| 归档页面 | 按月份分组展示所有文章 |

> 详细部署说明见 [.github/hugo/README.md](./.github/hugo/README.md)

### 快速部署步骤

1. **修改配置文件**
   - 编辑 [.github/hugo/hugo.toml](./.github/hugo/hugo.toml)，将 `baseURL` 修改为你的 GitHub Pages 地址
   - 修改 GitHub 菜单链接为你的仓库地址

2. **初始化主题**
   ```bash
   git submodule add https://github.com/adityatelange/hugo-PaperMod.git .github/hugo/themes/PaperMod
   ```

3. **启用 GitHub Pages**
   - 仓库设置 → Pages → Source 选择 "GitHub Actions"

4. **提交代码**
   ```bash
   git add .
   git commit -m "chore: add hugo deployment"
   git push origin main
   ```

5. **访问网站**
   部署完成后访问 `https://yourusername.github.io/hacknews/`
