## 🎯 核心内容/论点

这是一份面向 **Shell 重度用户（bash/zsh/fish）** 的实用技巧合集。文章不是入门教程，而是**中高级技巧的精选集合**——那些能显著提升日常终端工作效率、减少重复劳动、避免常见陷阱的"小技巧"。

### 🔧 核心技巧分类

#### 1️⃣ 命令行效率提升

```bash
# !! — 重复上一条命令
sudo !!

# !$ — 上一条命令的最后一个参数
mkdir new_folder && cd !$

# ^old^new — 快速替换上一条命令中的文本
^gcc^clang

# Ctrl+R — 反向搜索历史（增量搜索）
# Ctrl+S — 正向搜索历史（需先执行 stty -ixon）

# Alt+. — 循环遍历上一条命令的参数
```

#### 2️⃣ 文件操作技巧

```bash
# brace expansion — 批量创建/操作文件
cp config.{prod,dev}.yaml backup/
mv image{,.bak}

# process substitution — 避免临时文件
diff <(curl -s url1) <(curl -s url2)

# 同时重命名多个文件（使用 rename 或 zmv）
zmv '(*).log' '$1.txt'

# 查找并安全删除
find . -name "*.tmp" -mtime +7 -delete
```

#### 3️⃣ 流处理与管道

```bash
# tee — 分流输出到文件和屏幕
command | tee output.log | grep "error"

# 使用 /dev/stderr 分离正常和错误输出
{ command 2>&1 1>&3- | grep "error"; } 3>&1

# pv — 带进度条的管道
tar czf - /large/dir | pv | ssh remote "cat > backup.tar.gz"

# parallel — 并行处理
find . -name "*.jpg" | parallel --jobs 4 convert {} {.png}
```

#### 4️⃣ 调试与排错

```bash
# set -x — 打印每条执行的命令（调试模式）
set -x; complex_command; set +x

# trap — 捕获信号和错误
trap 'echo "Error at line $LINENO"' ERR

# 查看命令实际执行了什么（不真正执行）
bash -n script.sh    # 语法检查
bash -v script.sh    # 显示输入
bash -x script.sh    # 追踪执行

# strace/ltrace — 系统调用/库函数追踪
strace -e trace=network command
```

#### 5️⃣ 终端体验优化

```bash
# cd — 自动跳转到最近使用的目录（需要 zoxide/z）
z recent_project

# fzf — 模糊搜索一切
cat large_file | fzf        # 搜索文件内容
kill -9 $(ps aux | fzf)     # 搜索并杀死进程
cd $(find ~ -type d | fzf)   # 模糊切换目录

# bat — 比 cat 更好的 cat（语法高亮 + Git 集成）
bat main.go

# exa/eza — 更好的 ls（颜色、图标、Git 状态）
eza --long --git --icons
```

### 🎯 文章亮点：那些"救命"的技巧

| 场景 | 技巧 | 效果 |
|------|------|------|
| 刚执行了需要 sudo 的命令 | `sudo !!` | 不用重新输入 |
| 路径中有空格或特殊字符 | 引号 + tab 补全 | 避免解析错误 |
| 需要重复上条命令的一个参数 | `Alt+.` | 秒级操作 |
| 管道中间想看数据 | 插入 `tee` 或 `less` | 不破坏管道 |
| 大量文件的批量重命名 | brace expansion | 一行搞定 |
| 远程服务器卡住了 | `~.` （SSH escape）| 不用开新连接 |

## 💡 关键洞察 / 为什么重要

### ⚡ **Shell 是开发者的"母语"**

无论你用什么 IDE、什么语言、什么框架，**Shell 始终是你与系统交互的最底层接口**。

掌握 Shell 技巧的投资回报率极高：
- **一次学习，终身受用**（Shell 语法几十年没大变）
- **跨平台通用**（Linux/macOS/WSL 都能用）
- **可组合性无限**（每个工具都是管道的一环）
- **远程必用**（SSH 连接后只有 Shell）

### 🧠 **从"会用"到"熟练"的差距**

大多数开发者处于"够用"水平：
- 知道 `cd`、`ls`、`grep`、`pipe`
- 会基本的文件操作
- 遇到复杂情况就 Google

"熟练"水平的标志：
- **肌肉记忆**：常用快捷键不需要思考
- **组合思维**：自然地将小工具组合解决大问题
- **调试直觉**：知道哪里可能出问题以及如何排查
- **效率意识**：能感知哪些操作可以自动化

### 📈 **隐藏的时间成本**

一个每天节省 30 秒的 Shell 技巧：
- 每天 30 秒 × 250 工作日 = **每年 2 小时**
- 如果你有 20 个这样的技巧 = **每年 40 小时**
- 相当于**多出一周的工作时间**
