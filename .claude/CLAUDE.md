# HackNews Digest - 项目指令

## 项目目标

Hacker News 热点新闻**采集 → 深度解读 → 归档**系统。

### 三大目标
1. **采集**: 拉取指定日期范围 HN 每日 Top 10，中文翻译 + emoji 解读
2. **解读**: 对选中文章产出深度分析（技术细节、核心洞察、HN 社区评论）
3. **归档**: 结构化存储为月目录 + 周索引 + 独立文章文件

### 质量标准
- 标题必须中英双语（中文翻译 + 英文原文括号内保留）
- 解读必须以 emoji 开头，一句话概括
- 深度解读包含：核心论点、关键洞察、HN 评论精选、推荐资源
- 周度索引包含：主线话题、新兴趋势、一句话总结

---

## 用户操作流程（仅 2 步）

### 第一步：收集新闻

用户发送：
> 搜索 Hacker News 上 **[起始日期]** 到 **[结束日期]** 的热点新闻，每天要有 **[N]** 条

Agent 执行：
- 从 HN Daily / HN Algolia API 拉取数据
- 返回带编号的列表（`W.D` 格式）
- 标题翻译成中文，每条配 emoji 解读和热度数据

---

### 第二步：深度解读 + 归档（一站式完成）

#### 方式 A：指定编号

> 对 #1.3、#2.7、#3.5 这几篇做深度解读

#### 方式 B：指定标题

> 对这几篇做深度解读：
> 1. 认知暗森林
> 2. Copilot PR 广告事件

#### 方式 C：不指定（默认全部）

> 对以上所有文章做深度解读

**⚠️ 如果第二步没有指定具体文章，默认对第一步返回的全部文章进行深度解读。**

Agent 收到第二步指令后，**一次性完成以下全部工作**：

```
第二步内部执行流程:
┌───────────────────────────────────┐
│ ① 深度解读：逐篇撰写分析文章       │
│    → 遵循标准模板（核心论点/洞察/评论）│
├───────────────────────────────────┤
│② 归档写入：生成 YYYYMMdd-标题.md   │
│    → 写入 hacknews/YYYYMM/ 目录     │
├───────────────────────────────────┤
│③ 生成周度索引 weekN.md            │
│    → 按自然周拆分，含趋势总结       │
├───────────────────────────────────┤
│④ 更新 README.md                   │
│    → 更新统计、导航表、文章清单      │
├───────────────────────────────────┤
│⑤ 全项目检查                       │
│    → 检查遗漏、一致性、链接有效性    │
└───────────────────────────────────┘
```

---

### 清理模式（可选）

后续用户可能说：
> 清理部分文档 / 删除不必要的部分

此时 Agent 应当：
- **可以删除** README.md 中的文章索引条目（已读过的、不需要的）
- **不要动** weekN.md 中的列表内容（保持原始记录完整性）
- **可以删除** 不再需要的深度解读 .md 文件（需要用户指定, 或者用户手动删除文件, 不是自动删除）
- 清理后同步更新 README.md 统计数字

## 第一步详细规范

### 输出格式

```markdown
## 📅 YYYY年M月D日（周X）

| # | 标题 | 解读 | 热度 | 链接 | 来源 |
|---|------|------|------|------|------|
| W.D | **中文标题 (English Title)** | 🎯 一句话emoji解读 | XXX points | [平台](URL) | 数据源 |
```

### 编号规则
- 格式 `W.D` — **W** = 第几天(从1开始)，**D** = 当天第几条(1-10)
- 例：`1.1`=第1天第1条、`3.7`=第3天第7条

### 字段要求

| 字段 | 规则 |
|------|------|
| 标题 | 必须翻译中文，英文原标题保留在括号内 |
| 解读 | 必须以 emoji 开头，一句话概括 |
| 热度 | HN points（不确定时用 `~XXX`） |
| 链接 | `[平台简称](URL)` 格式 |
| 来源 | HN / HN Daily / BBC / NYT / arXiv / EFF 等 |

### Emoji 参考表

| 类别 | 可用 Emoji |
|------|-----------|
| 安全/隐私 | 🔒 ⚠️ 🛡️ 🚨 🕵️ |
| AI/技术 | 🤖 ✨ 🧪 🔄 💡 🔍 🎮 |
| 航天/硬科 | 🚀 🌍 🛰️ 📡 💾 🌳 |
| 商业/经济 | 💰 📊 🏛️ 💼 ❌ |
| 人文/社会 | 📖 🌲 😔 😤 👓 |
| 正面/创新 | ✅ 🎉 🌟 🆙 🏆 |
| 争议/警告 | ⚠️ 🚨 ☠️ ⛔ |

### 其他规则
- 每天固定数量条目（默认10条）
- 必须包含热度数据
- 按日期分组展示
- 按照热度排序

## 第二步详细规范

### 深度解读模板

```markdown
# Emoji [中文标题] (English Title)

**日期**: YYYY-MM-DD
**原文链接**: [平台名](URL)
**HN 热度**: XXX points | XX comments

---

## 🎯 核心内容/论点
（详细解读，包含技术细节、背景知识、社区反应等）

## 💡 关键洞察 / 为什么重要
（提炼核心价值和对行业的启示）

## 💬 HN 社区精彩评论
（摘录3-5条有代表性的讨论）

## 📎 推荐资源 / 延伸阅读（可选）
（书籍、工具、相关文章）
```

### 格式要求
- 标题行加一个主题 emoji
- 各章节标题加相关 emoji（`## 🎯`、`## 💡`、`## 💬`）
- 正文使用**中文**，专业术语可保留英文
- 代码块使用对应语言标签

---

## 周度索引规范 (weekN.md)

### 结构

```markdown
# Hacker News 每周热点 Top 10 — Week N（M月D日 - M月D日）

> 统计周期: YYYY-M-D 至 YYYY-M-D | 共X天，每日Top 10

---

## 📅 YYYY-MM-DD（周X）
| # | 标题 | 解读 | 热度 | 链接 | 来源 |
...（每天10条）

---

## 🔥 本周趋势总结

### 三大主线话题
| 主题 | 占比 | 关键事件 |

### 📈 新兴趋势
1. **🔢 趋势一** — 描述
2. ...

### 🎯 一句话总结本周
> **核心洞察**

---
> 📌 数据来源 | 🗓️ 统计周期 | 📝 深度解读归档链接
```

### 趋势总结要求
- 提炼 **2-4 个主线话题**，标注占比和关键事件
- 列出 **3-5 个新兴趋势**
- 用 **一句话** 总结本周核心洞察
- 风格简洁有力

## 文件命名与存放

### 命名规则
```
YYYYMMdd-中文标题-EnglishTitle.md
```
- 前缀：`YYYYMMdd`（8位日期，文章发布日期）
- 分隔符：`-`
- 标题：中文标题 + `-` + 英文标题（URL友好）
- **文件名不含 emoji**（emoji 仅在内容/标题行中出现）
- 扩展名：`.md`
- 存放位置：`hacknews/YYYYMM/`

### 目录结构

```
hacknews/
├── .claude/
│   └── CLAUDE.md          ← 本文件：项目指令（静态）
├── README.md              ← 项目状态导航（动态，每次归档后更新）
├── YYYYMM/
│   ├── week1.md           ← 该月第1周 Top10 索引
│   ├── week2.md           ← 该月第2周 Top10 索引
│   └── YYYYMMdd-中文-English.md  ← 深度解读文章
```

### 周拆分规则
- 按**自然周**拆分（周一-周日 或 周日-周六，保持一致）
- 跨月的周按实际日期拆到对应月份文件夹，并注明关联
- 每月周编号从 `week1` 开始递增

## 数据源

| 数据源 | 用途 | 链接 |
|--------|------|------|
| **HN Daily (daemonology)** | 每日 Top 10 归档（**首选数据源**） | https://www.daemonology.net/hn-daily/ |
| **HN Algolia API** | 精确搜索文章详情（points/comments/URLs） | https://hn.algolia.com/api/v1/search?query=... |
| **HN 首页** | 实时热门 | https://news.ycombinator.com/ |
| **HN Best** | 历史最高票 | https://news.ycombinator.com/best |
| **SuperTechFans** | 中文译介补充 | https://www.supertechfans.com/cn/post/ |

### 采集策略
1. **首选**：HN Daily 获取每日 Top 10 列表（最可靠的归档源）
2. **补充**：WebSearch 补充缺失元数据（points、URLs、精确标题）
3. **兜底**：HN Algolia API 按标题/关键词精确查找
4. **交叉验证**：多源数据冲突时交叉核对

## 第二步完成后的检查清单

归档完成后，Agent 必须自行检查以下项：

### 内容质量
- [ ] 所有标题都有中文翻译 + 英文原文
- [ ] 所有解读都以合适 emoji 开头
- [ ] 每条都包含热度数据
- [ ] 所有链接格式正确且有效
- [ ] 深度解读遵循标准模板
- [ ] 周度索引包含趋势总结
- [ ] 文件名不含 emoji

### 一致性
- [ ] weekN.md 的条目与第一步返回的列表一致
- [ ] 深度解读文件名符合命名规范
- [ ] 深度解读文件的日期与实际发布日期匹配
- [ ] README.md 统计数字已更新（总天数、条目数、解读数）

### 完整性
- [ ] 所有指定的深度解读文章都已生成
- [ ] 所有月份目录都已创建对应的 weekN.md
- [ ] README.md 导航表中包含新文章链接
- [ ] 无孤立文件（每个 .md 都被 weekN.md 或 README 引用）

### 输出报告

完成后向用户汇报：

```markdown
**周度索引**:
- hacknews/YYYYMM/weekN.md（X天，Y条新闻）

- YYYYMMDD-标题1.md
- YYYYMMDD-标题2.md
- ...

**README.md 已更新**: 统计数据 / 导航表 / 文章清单

📋 检查结果: 无异常 / 发现 X 处需注意的问题
```
