## 🎯 核心内容/论点

这是一篇**关于 AI API 匿名请求令牌（Anonymous Request Tokens, ART）的技术安全分析**。研究者发现：不同版本的 Claude Opus 模型在处理匿名化/隐私保护请求时的 Token 消耗模式存在显著差异，这些差异可能泄露用户意图信息。这项研究对 AI 隐私保护、API 安全审计和模型行为分析具有重要意义。

### 🔬 技术细节：

**什么是 Anonymous Request Tokens (ART)?**

```
ART System Overview:

传统 API 请求:
User → [Identity + Query] → API Server → Model → Response
问题: API Provider 可以看到用户身份 + 查询内容

ART 化请求:
User → [Anonymized Identity + Query + Noise Injection] → API Server → Model → Response
目标: API Provider 无法将查询关联到特定用户
```

**Opus 版本间的差异来源**:

```python
# 不同 Opus 版本处理 ART 的伪代码差异

class Opus3_ART_Handler:
    def process(self, request):
        # 版本 3: 基础实现
        identity = hash(request.user_id)  # 可逆哈希!
        query = request.query_text
        return self.model.generate(identity + query)
        # 问题: 模型可能从 hash 反推用户特征

class Opus35_ART_Handler:
    def process(self, request):
        # 版本 3.5: 改进版
        identity = secure_hash(request.user_id + salt)  # 加盐哈希
        noise = generate_noise(epsilon=0.1)
        query = add_differential_privacy(request.query_text, noise)
        return self.model.generate(identity + query)
        # 改进: 但噪声可能被模型"学习去除"

class Opus4_ART_Handler:
    def process(self, request):
        # 版本 4: 架构级改进
        identity = None  # 完全剥离!
        query = request.query_text
        context_window = isolate_user_context(request)  # 上下文隔离
        return self.model.generate(query, isolated_context=True)
        # 更好: 但仍有 token 消耗模式的侧信道

class Opus47_ART_Handler:  ★ 最新版本
    def process(self, request):
        # 版本 4.7: 全面隐私保护
        identity = None
        
        # 1. 差分隐私注入 (formal guarantees)
        query = dp_mechanism(request.query_text, epsilon=0.05, delta=1e-5)
        
        # 2. Token 归一化 (消除长度侧信道)
        normalized = normalize_token_count(query, target=850, variance=50)
        
        # 3. 时序混淆 (消除时间侧信道)
        delayed = timing_obfuscation(normalized, delay_range=(100ms, 500ms))
        
        # 4. 输出过滤 (防止训练数据泄露)
        response = self.model.generate(delayed)
        filtered = pii_scrubber(response, strict_mode=True)
        
        return filtered
```
