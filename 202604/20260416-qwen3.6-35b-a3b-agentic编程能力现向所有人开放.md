### 🚀 Agentic 能力详解：

**什么是 "Agentic Coding"?**

传统代码生成 (Code Completion):
```
User: 写一个 Python 函数计算斐波那契数列
Model: def fib(n): ...  (一次性输出)
```

Agentic 编程 (Agentic Coding):
```
User: 帮我重构这个项目的认证模块，使其支持 OAuth2 + SSO + MFA

Model (Agent):
  Step 1: 分析项目结构和现有认证代码 (多文件读取)
  Step 2: 设计新的认证架构 (规划)
  Step 3: 创建 OAuth2 provider 配置文件 (文件创建)
  Step 4: 实现 SSO 登录端点 (代码编写)
  Step 5: 添加 MFA 中间件 (安全增强)
  Step 6: 编写单元测试 (质量保证)
  Step 7: 运行测试并修复失败的 case (调试循环)
  Step 8: 更新文档 (文档同步)
  
  全程自主执行，仅在需要确认时询问用户
```

### 🛠️ 技术创新点：

**1. Sparse Expert Routing 的优化**

```python
# Qwen3.6 的 MoE 路由策略 (简化版)

class QwenMoE(nn.Module):
    def __init__(self):
        self.experts = nn.ModuleList([Expert() for _ in range(32)])
        self.gate = Router()  # 学习到的路由器
        self.shared_expert = SharedExpert()  # 共享专家 (始终激活)

    def forward(self, x):
        # Step 1: 共享专家处理 (捕获通用模式)
        shared_out = self.shared_expert(x)

        # Step 2: 路由决策 (只选 top-2 最相关的专家)
        router_logits = self.gate(x)
        top_k_indices = torch.topk(router_logits, k=2).indices

        # Step 3: 稀疏激活 (只计算选中的专家)
        expert_outputs = []
        for i in range(2):
            expert_idx = top_k_indices[:, i]
            selected_expert = self.experts[expert_idx]
            expert_outputs.append(selected_expert(x))

        # Step 4: 加权组合
        sparse_out = sum(expert_outputs) / 2

        return shared_out + sparse_out  # 残差连接
```

**为什么这样设计高效?**
- **共享专家**: 捕获所有 token 都需要的通用语言能力（语法、基础逻辑）
- **稀疏专家**: 只在需要时激活专业化的知识（特定领域的编程模式）
- **结果**: 大多数情况下只需 3B 参数的计算量，但拥有 35B 参数的知识容量

**2. 代码专用的训练策略**

训练流程的创新：
```
Phase 1: Pre-training (基础预训练)
├── 数据: 15T tokens (78% code, 22% natural language + math)
├── 重点: 多语言代码覆盖 (200+ 编程语言)
├── 特色: 包含完整的 GitHub 仓库 (不仅是单个文件)
└── 时长: 35 天 (8K H100)

Phase 2: Agentic Post-training (Agent 后训练) ★ 核心差异化
├── 数据: 500M tokens 的 agent trajectory data
│   ├── 成功的 agent 执行轨迹 (来自内部工具使用记录)
│   ├── 失败的轨迹 + 修正 (学习从错误中恢复)
│   ├── 人工标注的高质量 agent 行为
│   └── 合成的复杂多步编程任务
├── 方法: 
│   ├── Online DPO (Direct Preference Optimization)
│   │   ——偏好成功的 agent 行为模式
│   ├── Process Reward Modeling
│   │   ——奖励正确的中间步骤（不仅看最终结果）
│   └── Tool-use fine-tuning
│       ——强化 Bash/Git/文件操作的能力
└── 时长: 10 天 (4K H100)

Phase 3: Safety Alignment (安全对齐)
├── 拒绝有害代码生成 (malware, exploits)
├── 版权代码的去污染 (避免 verbatim reproduction)
├── 安全工具使用的约束 (不执行危险命令)
└── 时长: 3 天 (2K H100)
```

**3. 长上下文优化**

128K context 的实现：
- **RoPE 扩展**: 使用 YaRN (Yet another RoPE extension) 支持 128K
- **Attention Sink Tokens**: 在开头添加可学习的 sink tokens 稳定长序列 attention
- **Cache Optimization**: PagedAttention + FlashAttention-3 混合策略
- **实际效果**: 在 100K tokens 的代码库上仍保持 >95% 的检索准确率


**部署挑战**:
- **VRAM 要求**: 24GB 显存是门槛 (虽然比之前的 80GB 低很多)
